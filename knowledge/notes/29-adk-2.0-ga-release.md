# ADK 2.0 GA Release

## Overview

ADK (Agent Development Kit) 2.0.0 was released for general availability
on May 19, 2026, introducing a major architectural shift in how AI
agents are built and orchestrated ([ADK 2.0 announcement](https://adk.dev/2.0/)).

The new version focuses on deterministic control, predictability, and
reliability for complex multi-agent workflows.

## Key Features

### Graph-Based Workflows

Deterministic agent workflows with controlled task routing and execution
([Graph routes](https://adk.dev/graphs/routes/)). Supports:

- Route-based decision paths for directing agent execution
- Data handling between workflow steps
- Human-in-the-loop input points
- Dynamic graph generation during execution

### Dynamic Workflows

Code-based logic for building complex workflows including:

- Iterative loops with configurable conditions
- Complex decision-based branching
- Dynamic workflow generation at runtime

### Collaborative Workflows

Multi-agent architectures with:

- Coordinator agents delegating to sub-agents
- Sequential agents for pipeline-style processing
- Parallel agents for concurrent task execution
- Loop agents for iterative refinement

## Architecture Primitives

ADK is built around these core concepts ([About ADK](https://adk.dev/get-started/about/)):

- **Agent**: Fundamental worker unit; uses LlmAgent for reasoning or
  acts as deterministic controller via WorkflowAgent variants.
- **Tool**: Extends agent capabilities beyond conversation (APIs,
  search, code execution, external services).
- **Callbacks**: Custom code at specific agent lifecycle points.
- **Session & State**: Conversation context, history (Events),
  and working memory per session.
- **Memory**: Cross-session context for long-term recall.
- **Artifact**: Versioned file and binary data management.
- **Code Execution**: Agent-generated code execution capability.
- **Planning**: ReAct-style decomposable goal planning.
- **Models**: Underlying LLM powering reasoning (Gemini, Gemma,
  Anthropic, Ollama, APIee, and Agent Platform).
- **Event**: Basic unit of communication (user message, agent reply,
  tool use) forming conversation history.
- **Runner**: Execution flow engine orchestrating agent interactions.

## Runtime and Developer Tooling

- **Agent Runtime**: Supports all listed languages.
- **Streaming**: Native bidirectional streaming (text and audio) via
  Gemini Live API Toolkit.
- **Evaluation**: Built-in multi-turn evaluation datasets, CLI and
  Dev UI support.
- **CLI & Dev UI**: Local development, execution step inspection,
  debugging, and agent definition visualization.
- **Extensibility**: Open ecosystem for third-party tools and data
  connectors.
