covers "Setting up a Model Context Protocol (MCP) server." This is distinct from
the MCP server reference page (06) and would serve as a practical walkthrough
for the target audience.

**Audience impact:** Engineers evaluating MCP setup need a step-by-step guide.

### 2. Examples (`docs/get-started/examples.html`) — HIGH PRIORITY

Contains four concrete usage examples:

- Rename photographs based on content
- Explain a repository by reading its code
- Combine two spreadsheets into one spreadsheet
- Run unit tests

**Audience impact:** Practical examples are essential for adoption and comparison
with competing tools (Claude Code, Codex). Currently no examples exist in the
knowledge base.

### 3. Troubleshooting (`docs/troubleshooting.html`) — HIGH PRIORITY

Covers:

- Authentication or login errors (e.g., "Request contains an invalid argument")
- Common error messages (e.g., EADDRINUSE for MCP servers)
- Exit codes (e.g., 41 = FatalAuthenticationError)
- Debugging tips (CLI --verbose flag, core debugging)
- Existing GitHub issues and creating new issues

**Audience impact:** Critical for users adopting the tool. The FAQ note (15)
covers some error codes but not the full troubleshooting taxonomy.

need to understand data handling and privacy implications.

### 5. NPM Package Overview (`docs/npm.html`) — MEDIUM PRIORITY

NPM workspaces. Relevant for developers evaluating or contributing to the project.

important notes on binary file handling. The tools note (03) mentions individual

### 7. Web fetch tool (`docs/tools/web-fetch.html`) — MEDIUM PRIORITY

Covers `web_fetch` tool with description, usage examples, and important notes on
URL processing. The tools note (03) lists web-fetch but does not go into depth.

### 8. Web search tool (`docs/tools/web-search.html`) — MEDIUM PRIORITY

Covers `google_web_search` tool with description, usage examples, and important

### 9. Shell tool (`docs/tools/shell.html`) — MEDIUM PRIORITY
